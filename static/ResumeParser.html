Overview

The Resume Parser System automates the extraction of key information from resumes in formats like PDF, DOCX, or TXT. When a user uploads a resume, the API sends it to a Text Extractor, which reads and extracts its content. If the text is successfully extracted, the Named Entity Recognition (NER) Model identifies important details such as the candidateâ€™s name, experience, education, and skills, while a Regex Parser extracts contact information like email, phone number, and LinkedIn profile. The structured data is then stored in a database, and the API returns a JSON response containing the parsed details to the user. If the text extraction fails, the system logs the error in the database and informs the user with an appropriate error message

System Architecture

  1. Text Extraction

    • Use PyMuPDF, pdfplumber, or Tika for PDFs
    • Use python-docx for DOCX files
    • Use Tesseract OCR for scanned documents

  2. Information Extraction

    • Named Entity Recognition (NER) using SpaCy or BERT-based models
    • Use regex-based parsing for structured patterns (e.g., emails, phone numbers)

  3. Skills & Keyword Matching

    • Implement TF-IDF, Word2Vec, or BERT embeddings
    • Normalize skills using an ontology (e.g., O*NET or a predefined skills database)

  4. Experience & Education Parsing

    • Detect job titles, company names, and employment dates
    • Recognize degree names and university names

  5. Contextual Understanding

    • Fine-tune a BERT/GPT model for better parsing accuracy

  6. Scalability & API Integration

    • Develop a RESTful API using FastAPI or Flask
    • Deploy on AWS Lambda, GCP, or Azure
